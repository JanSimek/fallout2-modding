# Engine API Overview

This section documents the internal API of the Fallout 2 Community Edition engine, helping modders and contributors understand how the engine works.

## Architecture

The fallout2-ce engine is written in C++ and organized into modules:

| Module | Description |
|--------|-------------|
| `interpreter` | SSL bytecode execution |
| `scripts` | Script management and events |
| `proto` | Prototype loading and caching |
| `object` | Game object management |
| `critter` | Critter-specific logic |
| `item` | Item handling |
| `combat` | Combat system |
| `map` | Map loading and rendering |
| `tile` | Tile-based positioning |
| `art` | Graphics and animation |
| `interface` | UI elements |
| `dialog` | Dialogue system |

## Script Opcodes

SSL scripts call engine functions through opcodes. Each opcode has a handler function:

```cpp
// Example opcode handler signature
void opGetCritterStat(Program* program);
```

## Object System

All game entities (items, critters, scenery, etc.) are represented as `Object` structures:

```cpp
typedef struct Object {
    int id;
    int tile;
    int elevation;
    // ... many more fields
} Object;
```
